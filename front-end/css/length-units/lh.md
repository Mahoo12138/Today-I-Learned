## 一、背景：Web 排版的痼疾

在 Web 上创造精美的排版其实已经可以做到几十年了，但要把细节打磨到极致往往需要花费大量的精力——多到开发者根本没有时间去做。

好消息是，近几年 CSS 陆续推出了许多小而精的新工具，其中一个就是 **`lh` 单位**。

## 二、核心概念：`lh` 与 `rlh`

### 2.1 定义

| 单位  | 全称             | 含义                                                         |
| ----- | ---------------- | ------------------------------------------------------------ |
| `lh`  | Line Height      | **当前元素**一行文字的高度（受当前字体和 `line-height` 影响） |
| `rlh` | Root Line Height | **根元素**（`<html>`）一行文字的高度（类比 `rem` 与 `em` 的关系） |

一句话记忆：**`lh` 之于 `line-height`，正如 `em` 之于 `font-size`；`rlh` 之于 `lh`，正如 `rem` 之于 `em`。**

### 2.2 与现有单位的对比

```
em   → 当前元素的 font-size
rem  → 根元素的 font-size
lh   → 当前元素的 line-height（= font-size × 行高倍数）
rlh  → 根元素的 line-height
```

### 2.3 浏览器支持

- 所有主流浏览器均于 **2023 年**完成支持
- 截至 2025 年 04 月 **，覆盖率超过 **94%** 的用户浏览器

## 三、最佳实践：用 `lh` 设置段落间距

### 3.1 旧方式 vs 新方式

```css
/* 旧方式：自 90 年代就一直是浏览器默认值 */
p { margin-block: 1em; }

/* 新方式：与行高对齐，更精致 */
p { margin-block: 1lh; }
```

**为什么 `1lh` 更好看？**

当段落间距等于一行文字的高度时，视觉上形成了一套严整的「基线网格（Baseline Grid）」：

- 空白区域 = 恰好一行文字的位置
- 文字节奏感更强，版面更有呼吸感
- 印刷设计中早已普遍使用这一原则

### 3.2 视觉对比（理解网格）

```
[有行高网格时]

  第一段第一行 ──── 网格线
  第一段第二行 ──── 网格线
  ← 1lh 空白 → ──── 网格线（空行，正好占一行位置）
  第二段第一行 ──── 网格线
  第二段第二行 ──── 网格线
```

当使用 `1em` 时，由于 `em` 基于 `font-size` 而非 `line-height`，段落间距会略小于一整行，文字不再对齐网格线，产生轻微的错位感。

## 四、扩展应用场景

`lh` 可以用在任何接受 `<length>` 类型的 CSS 属性上，不仅限于 margin。

### 4.1 Padding

```css
article {
  padding: 2lh;
}
```

让容器的内边距也与文字节奏对齐，整体版面更协调。

### 4.2 配合 `margin-trim` 修剪多余空白

```css
article {
  padding: 2lh;
  margin-trim: block; /* 消除首尾段落的 margin，避免双重叠加 */
}

p {
  margin-block: 1lh;
}
```

**问题背景**：当 `padding: 2lh` 和 `margin: 1lh` 同时存在时，第一段/最后一段的 margin 会与 padding 叠加，产生 `3lh` 的超大间距。`margin-trim` 可以精确裁掉这部分多余空间。

### 4.3 固定行数的容器高度

```css
.preview-box {
  height: 3lh;        /* 精确显示 3 行文字 */
  overflow: hidden;
}
```

这比 `calc(3 * 1.5em)` 更直观、更可靠，因为它直接等于 3 行的实际高度。

### 4.4 动画：文字滚动效果

```css
.ticker {
  height: 1lh;
  overflow: hidden;
}

.ticker-inner {
  animation: scroll 3s linear infinite;
}

@keyframes scroll {
  from { transform: translateY(0); }
  to   { transform: translateY(calc(1lh - 100%)); }
}
```

> 来自社区实践：将容器高度设为 `1lh`，然后用 `translateY` 从 `0` 滚动到 `calc(1lh - 100%)`，可以确保最后一行内容在停止时完整显示。

### 4.5 Gap 与 Grid 布局

```css
.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1lh;  /* 卡片间距与正文行高一致 */
}
```

## 五、渐进增强：兼容旧浏览器

对于不支持 `lh` 的浏览器（约 6%），使用「先写 fallback，再覆盖」的写法：

```css
article {
  padding: 1em;  /* fallback：不支持 lh 的浏览器使用 */
  padding: 1lh;  /* 支持 lh 的浏览器会覆盖上面这行 */
}
```

CSS 的级联机制保证了：

- 支持 `lh` 的浏览器：采用 `1lh`
- 不支持 `lh` 的浏览器：忽略第二行，保留 `1em`

也可以使用 `@supports` 进行更明确的特性检测：

```css
@supports (padding: 1lh) {
  article {
    padding: 1lh;
    margin-block: 1lh;
  }
}
```

## 六、`lh` vs `em` vs `rem`：何时选哪个？

| 场景                         | 推荐单位      | 理由                       |
| ---------------------------- | ------------- | -------------------------- |
| 段落间距                     | `lh`          | 与行高对齐，形成视觉节奏   |
| 容器 padding（文本内容区域） | `lh` 或 `rlh` | 保持与文字节奏一致         |
| 字体大小相关的间距           | `em`          | 随字号缩放                 |
| 全局统一间距                 | `rem` / `rlh` | 全站一致，不受局部字号影响 |
| 固定行数的高度               | `lh`          | 精确匹配行数，无需计算     |
| 组件内的局部间距             | `em`          | 组件自包含，随组件字号变化 |

------

## 七、注意事项

1. **`lh` 反映的是「意图」，不是「实际盒子高度」** 当行内包含特殊字符（emoji、超高字形）时，实际渲染的行框可能略高于 `lh` 值。`lh` 始终等于 `line-height` 属性的计算值。
2. **`rlh` 在组件库中更稳定** 在设计系统或组件库中，使用 `rlh` 可以确保所有组件的间距参考同一个基准，不受组件内部字号变化的影响。
3. **与 `calc()` 搭配使用**

```css
   .sidebar {
     width: calc(20lh + 2rem);  /* 混合使用完全合法 */
   }
```

## 八、知识延伸：竖排排版中的 `lh`

在中文竖排排版（`writing-mode: vertical-rl`）场景中，`lh` 同样有效——此时「一行」变为竖向排列，`lh` 等于竖向文字的行宽，可用于设置竖排内容的列间距。

```css
.vertical-text {
  writing-mode: vertical-rl;
  column-gap: 1lh;
}
```

## 九、总结

| 要点         | 内容                                                         |
| ------------ | ------------------------------------------------------------ |
| `lh` 是什么  | 当前元素一行文字的高度                                       |
| `rlh` 是什么 | 根元素一行文字的高度                                         |
| 最核心的用法 | `p { margin-block: 1lh; }` 实现基线网格对齐                  |
| 浏览器支持   | 2023 年全面普及，94%+ 覆盖率                                 |
| 兼容写法     | 先写 `em` 后写 `lh`，利用 CSS 级联实现渐进增强               |
| 配合属性     | `margin-trim` 消除首尾多余间距                               |
| 适用范围     | 任何接受 `<length>` 的属性：margin、padding、gap、width、height…… |

------

> **一句话总结**：`lh` 单位让 Web 排版终于能像印刷设计一样，将空间节奏与文字节奏绑定在一起，用最少的代码实现最精致的版面效果。
